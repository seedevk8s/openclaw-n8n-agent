# 오픈클로(OpenClaw) + 디스코드 연동 설치 가이드

> 이 문서는 WSL2 설치부터 오픈클로 설치, 디스코드 봇 생성 및 실제 연동 테스트까지 순서대로 따라가면 완성되도록 작성되었습니다. 실제 진행 중 겪었던 문제와 해결 방법도 함께 포함되어 있습니다.

---

## 0. 사전 준비물

- Windows 10 (버전 2004 이상) 또는 Windows 11
- 관리자 권한 계정
- 디스코드 계정 (없으면 discord.com에서 이메일로 가입, 인증도 이메일만 필요해 안정적)

---

## 1. WSL2 설치

### 1-1. PowerShell을 **관리자 권한**으로 실행

시작 메뉴 → PowerShell 우클릭 → "관리자 권한으로 실행"

### 1-2. WSL 설치 명령 실행

```powershell
wsl --install
```

- 이 명령은 WSL2 기능 활성화 + 기본 배포판(보통 Ubuntu)까지 한 번에 설치합니다.
- 설치 후 **재부팅**이 필요할 수 있습니다.

### 1-3. Ubuntu 배포판 확인

재부팅 후 PowerShell에서:

```powershell
wsl -l -v
```

`Ubuntu` 또는 `Ubuntu-24.04` 같은 배포판이 `Running` 또는 `Stopped` 상태로 보이면 정상입니다.

### 1-4. 배포판 실행 및 초기 계정 설정

```powershell
wsl -d Ubuntu-24.04
```

처음 실행 시 유닉스 사용자명/비밀번호를 설정하라고 나옵니다. 원하는 대로 입력합니다.

> 💡 **참고**: 나중에 오픈클로를 여러 개 운영하고 싶다면, 용도별로 배포판을 따로 만들어 이름을 구분하는 것도 방법입니다 (예: `openclaw01`). 처음엔 기본 Ubuntu 배포판 하나로 충분합니다.

---

## 2. Node.js 설치 (WSL 안에서)

오픈클로는 Node.js 기반 CLI 도구입니다. WSL 프롬프트 안에서:

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

설치 확인:

```bash
node -v
npm -v
```

버전이 출력되면 정상입니다.

---

## 3. 오픈클로 설치

```bash
npm install -g openclaw
```

설치 후 버전 확인:

```bash
openclaw --version
```

## 4. 디스코드 봇 생성 (개발자 포털)

이 단계는 **오픈클로와 무관하게, 디스코드 쪽에서 먼저 준비**하는 작업입니다.

### 4-1. 애플리케이션 생성

1. 브라우저에서 **https://discord.com/developers/applications** 접속
2. 우측 상단 **"신규 애플리케이션"** 클릭
3. 이름 입력 (예: `TeamBot`, `OutboundBot` 등 — 용도가 드러나는 이름 권장) → 생성

### 4-2. 토큰 발급

1. 왼쪽 메뉴 **"Bot"** 클릭
2. **"토큰 초기화"** 클릭 → 확인 팝업에서 **"네, 할게요!"**
3. 표시된 토큰을 **즉시 복사**해서 안전한 곳(메모장 등)에 저장
   - ⚠️ 토큰은 이 화면을 벗어나면 다시 볼 수 없습니다. 잃어버리면 재발급해야 합니다.
   - ⚠️ 이 토큰은 비밀번호와 같습니다. 절대 공개 채팅, 커밋 메시지, 캡처 화면 등에 노출하지 마세요.

### 4-3. Intents 설정 (같은 Bot 페이지, 아래로 스크롤)

**Privileged Gateway Intents** 섹션에서:

- ✅ **Message Content Intent** (필수 — 꺼져 있으면 봇이 메시지 내용을 못 읽음)
- ✅ **Server Members Intent** (권장)
- Presence Intent는 꺼둔 채로 둬도 무방

체크 후 반드시 **저장(Save Changes)** 클릭.

### 4-4. 봇 초대 URL 생성

1. 왼쪽 메뉴 **"OAuth2"** 클릭
2. 아래로 스크롤 → **"URL 생성기(URL Generator)"** 섹션
3. **범위(Scopes)**: `bot` 체크
4. 체크하면 아래 **"봇 권한"** 섹션이 나타남 → 최소한 아래 항목 체크:
   - 메시지 보내기
   - 링크 임베드
   - 파일 첨부
   - 메시지 기록 보기
5. 생성된 URL을 복사

### 4-5. 서버에 봇 초대

1. 생성된 URL을 새 브라우저 탭에 붙여넣고 접속
2. **서버 선택** 드롭다운에서 초대할 서버 선택
   - 서버가 없다면, 먼저 디스코드 앱(discord.com/channels/@me)에서 왼쪽 하단 **"+"** → **"직접 만들기"** → **"저와 친구들을 위한 서버"**로 테스트 서버를 만들어둡니다.
3. **"인증하기"** 클릭 → 캡차 있으면 통과
4. **"성공!"** 화면이 뜨면 완료

> 💡 **채널 ID를 미리 확인해두면 나중에 편합니다.** 디스코드 앱에서 설정 → 고급 → **개발자 모드**를 켠 뒤, 실제로 봇이 응답할 채널을 우클릭 → **"채널 ID 복사"**로 미리 받아두세요. (아래 6단계 트러블슈팅에서 이유가 나옵니다.)

## 5. 오픈클로 온보딩 (WSL에서)

WSL 프롬프트에서:

```bash
openclaw onboard
```

마법사가 순서대로 진행됩니다. 각 단계에서 아래처럼 선택하세요.

| 화면 | 선택 |
|---|---|
| Select channel (QuickStart) | **Discord (Bot API)** ← 화살표로 정확히 이동했는지 확인 |
| How do you want to provide this Discord bot token? | **Enter Discord bot token** → 4-2에서 복사한 토큰 붙여넣기 |
| Configure Discord channels access? | **Yes** |
| Discord channels access | **Allowlist (recommended)** |
| Discord channels allowlist | 아래 참고 |
| Web search / Skills / Hooks 관련 질문들 | 전부 **Skip for now** (나중에 필요하면 `openclaw configure`로 추가 가능) |
| How do you want to hatch your agent? | **Hatch in Terminal (recommended)** |

### 허용목록(allowlist) 입력 시 주의사항 ⚠️

형식은 `서버이름/#채널이름` 처럼 입력하도록 안내되지만, **한글 채널명이나 카테고리가 있는 서버에서는 엉뚱한 채널(카테고리)로 잘못 매칭되는 경우가 실제로 있었습니다.**

**권장 방법**: 이름 매칭 대신, 4-4~4-5에서 미리 복사해둔 **실제 채널 ID**를 직접 사용하세요.

1. 일단 이름 형식(`서버이름/#채널이름`)으로 입력해서 온보딩을 완료합니다.
2. 온보딩이 끝난 후, 아래 명령으로 실제 어떤 채널로 잡혔는지 확인합니다:
   ```bash
   openclaw logs
   ```
   로그에서 아래와 같은 줄을 찾습니다:
   ```
   discord channels resolved: <서버ID>/<채널ID> (guild:서버이름; channel:채널이름)
   ```
3. `channel:` 뒤에 나온 이름이 내가 원했던 채널이 **맞는지 확인**합니다. 다르면(예: 채널이 아니라 카테고리 이름이 나옴), 아래처럼 직접 수정합니다:
   ```bash
   nano ~/.openclaw/openclaw.json
   ```
   `channels.discord.guilds.<서버ID>.channels` 아래에 있는 채널 ID를, 미리 복사해둔 **실제 채널 ID**로 바꿔줍니다. 또는 터미널에서 바로 치환:
   ```bash
   sed -i 's/잘못된채널ID/올바른채널ID/' ~/.openclaw/openclaw.json
   ```
4. 수정 후 게이트웨이 재시작:
   ```bash
   openclaw gateway restart
   ```
5. 다시 확인:
   ```bash
   openclaw channels status
   ```

## 6. 실제 대화 테스트

### 6-1. 디스코드에서 메시지 보내기

디스코드 앱에서 해당 서버의 해당 채널로 이동해서:

```
@봇이름 안녕
```

**반드시 봇을 멘션(@)해야 합니다.** 기본 설정(`requireMention: true`)상, 멘션 없는 평문 메시지는 무시하도록 되어 있습니다. (매번 멘션하기 번거로우면, `openclaw.json`의 해당 서버 설정에 `"requireMention": false`를 추가하면 되지만, 활발한 서버에서는 봇이 모든 대화에 반응하게 되니 주의가 필요합니다.)

### 6-2. 응답이 없을 때 확인 순서

```bash
openclaw channels status
```
→ `connected` 상태인지 확인

```bash
openclaw logs
```
→ 최근 로그에서 `discord channels resolved: ...` 줄의 `channel:이름`이 맞는지, 에러 메시지가 있는지 확인

## 7. 자주 겪는 문제 (트러블슈팅 요약)

| 증상 | 원인 | 해결 |
|---|---|---|
| 평문 메시지에 봇이 반응 안 함 | `requireMention` 기본값이 true | `@봇이름`으로 멘션해서 다시 시도 |
| 멘션해도 반응 없음 | 허용목록의 채널 ID가 실제 채널이 아니라 카테고리 등 엉뚱한 곳으로 잘못 매칭됨 | `openclaw logs`로 실제 매칭된 채널명 확인 → `openclaw.json`에서 올바른 채널 ID로 직접 수정 → 게이트웨이 재시작 |
| PowerShell에서 `openclaw` 명령이 안 먹힘 | WSL이 아니라 Windows PowerShell에서 실행함 | `wsl -d <배포판이름>`으로 WSL 안에 먼저 들어간 뒤 명령 실행 |
| WSL2를 재부팅했더니 봇이 죽어있음 | 게이트웨이가 systemd 서비스로 등록되어 있으면 자동 복구되는 경우가 많음 | `openclaw channels status`로 먼저 확인 → 안 되면 `openclaw gateway restart` |
| Discord 개발자 포털의 "봇 권한" 체크박스와 실제 초대 권한이 다른 것 같음 | Bot 페이지의 권한 체크박스는 참고용 계산기이고, 실제 초대 URL은 OAuth2 → URL Generator에서 별도로 생성됨 | OAuth2 탭에서 scope와 권한을 다시 체크해서 URL 생성 |

## 8. 보안 체크리스트 (팀에 배포하기 전 확인)

- [ ] 봇 토큰이 캡처 화면, 채팅 로그, 커밋 이력 등에 노출된 적이 있다면 **토큰 재발급**
- [ ] `openclaw.json`에 토큰이 평문으로 저장되므로, 이 파일을 깃 저장소 등에 실수로 커밋하지 않도록 `.gitignore` 확인
- [ ] 팀원마다 **각자 별도의 봇/토큰**을 발급받는 것을 권장 (하나의 토큰을 여러 명이 공유하지 않기)

## 부록: 자주 쓰는 명령어 모음

```bash
openclaw channels status      # 연결 상태 확인
openclaw logs                 # 실시간/과거 로그 확인
openclaw gateway restart      # 게이트웨이 재시작 (설정 변경 후 필요)
openclaw config get gateway.auth.token   # 게이트웨이 토큰 확인
openclaw models list --provider openai   # 사용 가능한 모델 목록
openclaw models set openai/gpt-5.4-mini  # 더 가벼운 모델로 변경
```
